Q2.1 — XML Declaration Rules + Document Structure

XML Declaration Rules & Document Structure with Example

XML Declaration Rules:

  • Must be the very first line in the XML file (no whitespace before it)
  • It is enclosed in <?xml ... ?>
  • version attribute is mandatory (usually "1.0")
  • encoding defines character set (default UTF-8)
  • standalone = "yes" means document doesn't rely on external DTD
  • All XML tags are case-sensitive
  • Every opening tag must have a corresponding closing tag
  • Tags must be properly nested
  • There must be exactly ONE root element
  • Attribute values must be enclosed in quotes

XML Document Structure: An XML document has two parts:

  • Prolog: XML declaration + optional DOCTYPE + processing instructions + comments
  • Document Body: Root element containing all child elements
<!-- XML Declaration (Prolog) -->
<?xml version="1.0" encoding="UTF-8"?>

Q5.1 — AJAX Architecture

AJAX Architecture with Steps and Diagram

AJAX architecture uses the XMLHttpRequest object as a bridge between client and server to exchange data without full page reloads.

Steps in AJAX:

  1. Event Occurs: User interaction (click, keypress) triggers event
  2. Create XHR: JavaScript creates an XMLHttpRequest object
  3. Open Request: xhttp.open("GET", "url", true) initialises connection
  4. Send Request: xhttp.send() sends request to web server
  5. Server Processes: Server processes request (DB query, etc.)
  6. Server Responds: Returns data (JSON/XML/HTML/Text)
  7. Callback: onreadystatechange fires when readyState=4 & status=200
  8. Update DOM: JavaScript updates the page with received data
Key Concept: Steps 1–4 and 7–8 happen on the client. Step 5–6 happens on the server. The entire process is asynchronous – the user can continue using the page during Steps 5–6.

Q2-A — Characteristics of XML

Characteristics of XML

  • Self-Descriptive: Data and its description are included together within tags
  • Extensible: You can create your own tags (unlike HTML which has fixed tags)
  • Platform Independent: XML works across all platforms and operating systems
  • Unicode Support: Supports international characters through UTF-8/UTF-16
  • Hierarchical Structure: Data is organized in tree structure with parent-child relationships
  • Strict Syntax: All tags must be properly opened, closed, and nested
  • Case-Sensitive: <Name> and <name> are different elements
  • Validates with DTD/XSD: Can be validated against a Document Type Definition or XML Schema
  • Data Separation: Separates data from presentation (unlike HTML)
  • Widely Supported: Supported by all major languages (Java, Python, PHP, JavaScript)
Q2-A (OR) — XML Syntax Rules: Tags, Elements, Attributes

Syntax Rules of XML Tags, Elements, and Attributes

Tag Rules:

  • Tags are case-sensitive: <Name> ≠ <name>
  • All tags must be properly closed: <tag>...</tag>
  • Self-closing tags are allowed: <br />
  • Tags must be properly nested (no overlapping)

Element Rules:

  • Must have exactly ONE root element
  • Element names cannot start with a number or contain spaces
  • Element names are case-sensitive
  • Elements can have text, child elements or both

Attribute Rules:

  • Attribute values MUST be in quotes (single or double)
  • An element cannot have two attributes with the same name
  • Attributes cannot contain child elements

Q1. What is XML? Write characteristics of XML?

XML (eXtensible Markup Language) is a flexible markup language for storing and transporting data. Unlike HTML, which has fixed tags, XML lets you define your own tags to describe data meaningfully.

Characteristics:

  • Self-Descriptive — tags describe the data
  • Extensible — you can define your own tags
  • Platform and software independent
  • Hierarchical tree-like structure
  • Case-sensitive
  • Strict syntax — all tags must be properly closed and nested
  • Supports Unicode (international characters)
  • Can be validated using DTD or XML Schema
  • Separates data from presentation

Q2. How to declare XML tag? Explain XML tags.

An XML tag is declared using angle brackets < >. Tags come in pairs — an opening tag and a closing tag (with a forward slash).

<!-- 2. End Tag: </name> -->
<!-- 3. Empty Tag: <br /> -->
<!-- 4. Tag with attribute: <tag attr="val"> -->

Q3. Rules of XML tags

  • All XML elements must have a closing tag
  • XML tags are case-sensitive (<Name> ≠ <name>)
  • XML elements must be properly nested (no crossing tags)
  • XML documents must have exactly one root element
  • XML attribute values must be quoted
  • Comments: <!-- comment -->
  • Special characters must use entities: & = &amp;, < = &lt;, > = &gt;
  • Tag names cannot start with a number or hyphen or contain spaces

Q1. What is AJAX?

AJAX (Asynchronous JavaScript And XML) is a technique that allows web pages to update parts of their content asynchronously by exchanging small amounts of data with the server behind the scenes. This means it is possible to update parts of a web page without reloading the whole page.

AJAX is NOT a programming language. It is a combination of: Browser's built-in XMLHttpRequest object + JavaScript and HTML DOM + CSS + JSON/XML for data.

Q2. XMLHttpRequest Objects

The XMLHttpRequest (XHR) is a built-in browser API object used to make HTTP requests from JavaScript. It is the foundation of all AJAX operations.

Key Properties: readyState (0–4), status (200, 404, etc.), responseText, responseXML, onreadystatechange

Key Methods: open(method, url, async), send(body), setRequestHeader(), abort()

var xhr = new XMLHttpRequest();
xhr.onreadystatechange = function() {
  if(xhr.readyState == 4 && xhr.status == 200) {
    console.log(xhr.responseText);
  }
};
xhr.open("GET", "api/data.json", true);
xhr.send();

Q3. Working of AJAX

  1. User Event: User performs an action (click, keypress, form submit)
  2. Create XHR: JavaScript creates an XMLHttpRequest object
  3. Configure: xhr.open(method, url, async) sets up the request
  4. Send: xhr.send() sends the request to the web server
  5. Server Processing: Web server processes the request (reads DB, etc.)
  6. Server Response: Server sends back a response (JSON, XML, text, HTML)
  7. Callback Fires: When readyState=4 and status=200, the callback runs
  8. DOM Update: JavaScript updates only the relevant part of the webpage
Key point: While steps 5–6 happen on the server, the user can still interact with the rest of the page. This is the "asynchronous" aspect of AJAX.

Q4. Features of AJAX

  • Asynchronous: Requests happen in background; no page freeze
  • No Full Page Reload: Only specific parts of DOM are updated
  • Reduced Server Load: Only data is transferred, not full HTML
